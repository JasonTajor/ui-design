# Engineering Constitution (copy into your repo root)

Protect the design system. If it's broken, fix it before building the feature.

1. Tokens only — spacing on the 8pt grid, semantic color tokens, typography roles.
   No arbitrary values.
2. No duplicate components — extend the primitive; don't fork it.
3. Primitives first — fix the token/primitive, then the component, then the feature.
4. Read the component source before trusting its spec.
5. Mobile-first and responsive by default (see the design-system skill's responsive.md).

The `design-system` plugin provides the team: /dashboard to build, /review to audit.
